Roadmap

The roadmap is an overview of the project direction. Detailed plans and priorities are tracked in milestones:

  • Version numbers (0.1, 0.2, …) track production releases. The next upcoming version has a (estimated) target date.
  • backlog holds low-priority items.
  • needs-owner holds zero-priority items (no feasible path to completion).

Note these are tentative and can be changed, moved, or dropped at any time.

Next

Concrete high-level feature areas and changes.

0.12+: "The year of the Nvim OOTB"

  • Prepare for 1.0
  • No more "Press Enter"
  • Plugin manager
  • packspec / pkg.json
  • Redesign --remote
  • File-change detection
  • multicursor, super-macros
  • Task abstraction, structured concurrency: vim.async
  • Unified event interface, nvim_on()

0.11

  • LSP "config" concept: lsp/* on 'runtimepath', vim.lsp.config(), vim.lsp.enable()
  • LSP: multiclient support
  • TUI: URL highlight, theme event
  • UI detach, :connect: #5035
  • stdlib: image API
  • Redesign (simplify) remote plugin concept, eliminate :UpdateRemotePlugins
    • Lua remote plugin host
  • Externalized UI: window layout events, messages
  • LSP: vim.lsp.server()
  • LSP: auto-completion (on_attach config)
